# 사업소득 그룹 수정 팝업(SPS_BI_08) - 최종 정책서

---

## 1. 화면 개요

- **화면 ID**: SPS_BI_08
- **화면명**: 사업소득 그룹 수정 팝업
- **목적**: 사업소득 월별 리스트 화면(SPS_BI_02)에서 예외 업종 합산 행을 클릭했을 때, 합산에 포함된 여러 건의 사업소득 데이터를 탭 기반으로 수정 또는 삭제하기 위한 팝업
- **종속 조건**: 해당 팝업은 예외 업종(940906/940907/940908)의 합산 그룹에 종속되며, 그룹 내 각 건은 서로 다른 지급연월을 가진다.
- **귀속 기준 예외 규칙**: SPS_BI_04(사업소득 수정 팝업)과 동일. 업종코드가 보험설계사(940906), 음료배달(940907), 방문판매원(940908) 중 하나이면서 **귀속연도 ≠ 지급연도**인 경우, 수정된 데이터는 **귀속연도 12월** 리스트에 표시된다.
  - 귀속연도 또는 업종코드를 수정할 경우 예외 규칙의 적용 여부가 변경될 수 있다.
  - **예외 규칙 적용 시 안내**: "수정" 버튼 클릭 시 저장 전에 confirm을 노출한다.
    - confirm 문구: **"예외 업종 데이터가 포함되어 있습니다. 귀속연도 12월에 표시됩니다."**
    - 확인 클릭 시: 전체 저장 진행
    - 취소 클릭 시: 저장 중단, 팝업 유지
- **진입 경로**: SPS_BI_02 화면의 합산 행 클릭 시 노출
- **연관 화면**:
  - SPS_BI_02 (사업소득 월별 리스트) — 부모 화면
  - SPS_BI_04 (사업소득 수정 팝업) — 개별 건 수정 팝업 (비합산 행용)

## 2. 요청한 화면 기능

### 2.1 탭 구조

- 팝업 상단에 **탭**을 제공한다. 각 탭은 합산 그룹 내 개별 레코드를 나타낸다.
- 탭 라벨: **"YYYY.MM 지급"** (해당 레코드의 지급연도.지급월)
  - 예: "2026.02 지급", "2026.03 지급", "2026.04 지급"
- 탭 정렬: 지급연월 오름차순
- 탭 클릭 시 해당 레코드의 편집 폼으로 전환한다.
- 유효성 에러가 있는 탭에는 **"!" 표시**를 추가하여 시각적으로 구분한다.
- **탭 오버플로우**: 탭 수가 많아 표시 영역을 초과할 경우, **드롭다운** 방식으로 나머지 탭을 선택할 수 있도록 한다.

### 2.2 닫기/이탈 방지 (공통 정책 3.3 적용)

- 팝업 외 영역 클릭, 닫기 버튼 클릭, 또는 **ESC 키** 입력 시 confirm 노출
- 단, 모든 탭에서 기존 입력값에서 변경된 내용이 없는 경우에는 confirm 없이 팝업을 바로 닫는다.
- confirm 문구: **"사업소득 그룹 수정을 취소하시겠습니까?"**

### 2.3 수정 완료 토스트 (공통 정책 3.4 적용)

- 수정 성공 시 토스트 1.5초 노출
- 문구: **"사업소득 그룹 수정을 완료했습니다."**
- 수정 완료 후 **팝업을 자동으로 닫고**, SPS_BI_02 리스트/카운트/상단 요약 정보를 최신 상태로 갱신한다.

### 2.4 탭별 입력 항목 정의

각 탭은 **독립적인 편집 폼**을 제공한다. SPS_BI_04(사업소득 수정 팝업)과 동일한 레이아웃을 사용한다.

| 항목명 | 입력 타입 | 수정 가능 여부 | 비고 |
|--------|----------|--------------|------|
| 귀속연도 | 단일 셀렉트박스 | **수정 가능** | 2025년 ~ 현재연도 |
| 귀속월 | 단일 셀렉트박스 | **수정 가능** | 1월 ~ 12월 |
| 성명(상호) | 단일 인풋박스 | **수정 가능** | 최대 50자, 허용 문자: 한글·영문·숫자·공백·허용 특수문자(&, ', -, ., ·, (, )), 공백만 입력 시 trim 후 빈값 판정(필수 에러). 허용되지 않은 문자는 입력 불가 |
| 내외국인 구분 | 라디오 버튼 | **수정 가능** | 내국인 / 외국인. 변경 시 세율 조건부 전환 트리거 |
| 주민(사업자)등록번호 | 단일 인풋박스 | **수정 가능** | 숫자만 입력 가능, 최대 13자리, 10자리 또는 13자리만 허용, 체크디짓 알고리즘 검증. 병의원(851101) 선택 시 10자리 이하만 허용. 마스킹 없이 전체 표시 |
| 업종코드 | 단일 셀렉트박스 | **수정 가능** | |
| 지급액 | 숫자 입력 필드 | **수정 가능** | 공통 정책 4.1(천 단위 콤마), 4.4(0 이상 정수만 허용) 적용 |

- **지급연월**: 탭 라벨에 표시되며, 수정할 수 없다. (각 탭의 고유 식별값)

### 2.5 자동 계산 영역

SPS_BI_04와 동일. 각 탭마다 독립적으로 자동 계산 결과를 표시한다.

| 항목명 | 표시형식 | 상태 | 비고 |
|--------|---------|------|------|
| 세율(%) | 퍼센트 | readonly (조건부 입력 가능) | 기본 3%, 조건에 따라 변동 |
| 소득세 | 금액(천 단위 콤마 + "원") | readonly | 공통 정책 4.1, 4.3 적용 |
| 지방소득세 | 금액(천 단위 콤마 + "원") | readonly | 공통 정책 4.1, 4.3 적용 |
| 실지급액 | 금액(천 단위 콤마 + "원") | readonly | 공통 정책 4.1, 4.3 적용 |

### 2.6 자동 계산 동작 규칙

- SPS_BI_04와 동일한 세율 결정 로직, 계산 순서, 소액부징수 판정을 적용한다.
- 각 탭은 독립적으로 계산한다.

### 2.7 유효성 검사 (공통 정책 2.3 적용)

- 검증 타이밍: **blur / submit**
- 에러 표시: 컴포넌트 하단에 에러 문구 노출
- 각 탭별로 독립적으로 유효성 검사를 수행한다.
- **유효성 에러와 수정 버튼 상태**: SPS_BI_04와 동일. 어느 탭이든 유효성 에러가 존재하면 **"수정" 버튼을 비활성화**하고, 모든 탭의 에러가 해소되면 재활성화한다.

#### 2.7.1 필수값 검증 (blur 시)

- 수정 가능 필수 입력 항목(귀속연도, 귀속월, 성명(상호), 내외국인 구분, 주민(사업자)등록번호, 업종코드, 지급액)이 미입력 상태에서 blur 시 에러
- **공백만 입력된 경우 trim 후 빈값으로 판정**하여 동일하게 필수 에러 처리
- 에러 문구: **"필수 입력 항목입니다."**

#### 2.7.2 귀속연월 유효성 검사 (blur 시)

- 귀속연월이 해당 탭의 지급연월보다 **이후** 날짜인 경우 에러
- 에러 문구: **"귀속연월은 지급연월보다 이전 날짜여야합니다."**

#### 2.7.3 주민(사업자)등록번호 자릿수 검사 (blur 시)

- 주민(사업자)등록번호는 **10자리 또는 13자리**만 유효하다.
- 10자리, 13자리가 아닌 경우 blur 시 에러 처리
- 에러 문구: **"주민(사업자)등록번호는 10자리 또는 13자리만 입력 가능합니다."**

#### 2.7.4 주민(사업자)등록번호 체크디짓 검사 (blur 시)

- 주민등록번호(13자리) 및 사업자등록번호(10자리)에 대해 **체크디짓 알고리즘 검증**을 수행한다.
- 체크디짓 검증 실패 시 에러 처리
- 에러 문구: **"유효하지 않은 주민(사업자)등록번호입니다."**

#### 2.7.5 병의원 업종 예외 검사 (blur 시)

- 업종코드가 **병의원(851101)**인 경우, 주민(사업자)등록번호는 **10자리 이하만** 허용
- 에러 문구: **"병의원인 경우, 사업자등록번호만 입력하실 수 있습니다."**

#### 2.7.6 중복 데이터 검사 (submit 시)

- 각 탭별로 **지급연월 + 귀속연월 + 주민(사업자)등록번호 + 업종코드**가 모두 동일한 데이터가 이미 존재하는 경우 (자기 자신 제외)
- 해당 탭에 에러를 표시한다.
- 에러 문구: **"지급연월, 귀속연월, 주민(사업자)등록번호, 업종코드가 동일한 사업소득이 존재합니다."**
- **참고**: 그룹 내 각 탭은 서로 다른 지급연월을 가지므로, 그룹 내 탭 간 중복은 발생하지 않는다.

### 2.8 일괄 검증 및 저장

- **"수정" 버튼** 클릭 시 **모든 탭**에 대해 일괄적으로 유효성 검사를 수행한다.
- 검증 실패 시 **첫 번째 에러가 발생한 탭**으로 자동 전환하고 에러를 표시한다.
- 모든 탭이 검증을 통과한 경우, **모든 탭의 데이터를 일괄 저장**한다.
- 일괄 저장은 탭 순서대로 수행한다.
- 저장 중 에러 발생 시 팝업을 유지하고 에러를 표시한다.

### 2.9 탭별 삭제 ("이 건 삭제")

- 각 탭에 **"이 건 삭제"** 버튼을 제공한다.
- 버튼 클릭 시 confirm 팝업 노출
  - 문구: **"YYYY년 M월 지급 건을 삭제하시겠습니까? 삭제한 정보는 복구할 수 없습니다."**
- confirm 확인 클릭 시:
  1. 해당 레코드를 **즉시 삭제** (서버 반영)
  2. 토스트 1.5초 노출 — 문구: **"사업소득 삭제를 완료했습니다."**
  3. 해당 탭을 팝업에서 **제거**
  4. 남은 탭 중 가장 가까운 탭으로 자동 전환
- **마지막 건 삭제**: 그룹 내 마지막 레코드를 삭제할 경우, 삭제 처리 후 **팝업을 자동으로 닫는다**.
  - SPS_BI_02 리스트/카운트/상단 요약 정보를 최신 상태로 갱신한다.
  - 토스트 1.5초 노출 — 문구: "사업소득 삭제를 완료했습니다."
- **삭제 후 수정**: 일부 탭을 삭제한 후 "수정" 버튼을 클릭하면, 남은 탭의 데이터만 저장한다.

### 2.10 그룹 분리

- 귀속연월 또는 업종코드를 수정하여 기존 합산 그룹의 그룹핑 키(귀속연월 + 주민등록번호 + 업종코드)가 변경된 경우, 해당 레코드는 저장 후 자연스럽게 기존 그룹에서 분리된다.
- 분리된 레코드는 월별 리스트에서 개별 행 또는 새로운 합산 그룹에 포함된다.
- 별도의 안내나 confirm 없이, 저장 후 리스트 갱신 시 자동으로 반영된다.
- **그룹 완전 해체**: 모든 탭의 그룹핑 키가 변경되어 기존 그룹에 속하는 레코드가 0건이 되는 경우도 허용한다. 저장 완료 후 팝업이 닫히고 SPS_BI_02 리스트를 새로고침한다.

### 2.11 안내 문구

팝업 하단에 다음 안내 문구를 **고정 표시**한다. SPS_BI_04(사업소득 수정 팝업)과 동일.

> ※ 지급총액 입력 시 업종코드에 따라 세율이 자동 적용되어 소득세, 지방소득세가 계산됩니다.
> ※ 소액부징수(소득세액이 1천원 미만)인 경우, 소득세가 면제됩니다.
> ※ 직업운동가(940904) 중 프로스포츠 구단과의 계약기간이 3년 이하인 외국인 직업 운동가일 경우, 세율 20%
> ※ 봉사료 수취자(940905) 중 「소득세법 시행령」제184조의2에 해당하는 봉사료 수입금액의 경우, 세율 5%

### 2.12 저장 실패 처리

- 저장(submit) 실패 시 (중복 검사 실패, 비즈니스 로직 실패 등) **팝업을 유지**하고 에러를 표시한다.
- 서버 에러: 공통 정책 5.1 alert 노출 후 팝업 유지

### 2.13 전체 동작 흐름

```
1. SPS_BI_02 합산 행 클릭 시 해당 그룹의 모든 레코드 데이터를 전달하며 그룹 수정 팝업을 호출한다.
2. 팝업 진입 시 탭이 생성되고, 각 탭에 기존 등록값이 바인딩된다.
3. 각 탭별로 독립적인 편집 폼과 자동 계산 결과를 표시한다.
4. 모든 필드(귀속연도, 귀속월, 성명(상호), 내외국인 구분, 주민(사업자)등록번호, 업종코드, 지급액)를 변경할 수 있다.
5. "이 건 삭제" 버튼으로 특정 탭의 레코드를 삭제할 수 있다. 마지막 건 삭제 시 팝업 닫힘.
6. "수정" 버튼 클릭 시 모든 탭에 대해 일괄 검증을 수행한다.
7. 검증 실패 시 첫 번째 에러 탭으로 자동 전환 + 에러 표시.
8. 예외 규칙 해당 시 confirm 노출 (취소 시 저장 중단).
9. 모든 조건 통과 시 전체 탭 데이터를 일괄 저장한다.
10. 저장 성공 시 토스트 노출 후 팝업을 닫고 부모 화면(SPS_BI_02)을 갱신한다.
```

### 2.14 서버 에러 처리 (공통 정책 5.1 적용)

- 서버 에러 발생 시 alert 팝업 노출
- 문구: **"서버 오류가 발생했습니다. 잠시 후 다시 시도해 주세요."**
