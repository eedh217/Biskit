# 🎯 목적

이 스킬은 **엑셀 업로드 공통 정책**을 기반으로 화면별 엑셀 업로드 기능 상세 기획문서를 자동 생성한다.

---

# 📋 정책 파일 참조

엑셀 업로드 정책의 상세 내용은 다음 파일을 참조한다:
- **위치**: `prototype/policy/common/excel-upload-policy.md`
- **내용**: 엑셀 업로드 버튼, 업로드 처리, 실패 처리, 데이터 저장 등 공통 정책

---

# 🚀 사용 방법

```
/excel-upload-policy [화면명] [저장 위치]

예시:
/excel-upload-policy 급여 엑셀 업로드 policy/PAYROLL/
```

**입력 요구사항**과 함께 실행하면, 공통 정책 + 화면별 요구사항을 결합하여 상세 기획문서를 생성한다.

---

# 📝 기획문서 생성 규칙

## 필수 확인 사항

각 화면은 반드시 아래 항목을 정의해야 한다:

1. **저장 모드** (신규 / 업데이트)
2. **업데이트 기준 컬럼** (업데이트 모드일 경우)
3. **row 실패 사유 정의**
4. **실패 리스트에 표시할 대표 식별값**
5. **엑셀 양식 파일 경로 및 구조**

정의되지 않으면 반드시 질문한다.

---

## 산출물 규칙

- 엑셀 업로드 기능 정의 시, 공통 정책 항목이 적용되었음을 체크리스트 형태로 포함한다.
- 화면별 추가 요구사항이 주어지면 공통 정책 + 추가 요구사항을 합쳐 최종 정책을 제시한다.
- 실패 처리 로직(부분 성공, 실패 다운로드)은 누락하지 않는다.

---

## 질문 규칙

불명확하거나 누락된 사항은 반드시 "질문 목록"으로 정리하여 제시한다.

### 질문이 필요한 대표 사례

1. **엑셀 양식**
   - 컬럼 구조가 정의되지 않은 경우
   - 필수/선택 컬럼이 명시되지 않은 경우

2. **저장 정책**
   - 저장 모드(신규/업데이트)가 명시되지 않은 경우
   - 업데이트 기준 컬럼이 정의되지 않은 경우

3. **실패 처리**
   - row 실패 사유가 정의되지 않은 경우
   - 대표 식별값이 정의되지 않은 경우

4. **유효성 검증**
   - 각 컬럼별 유효성 규칙이 명시되지 않은 경우

---

### 질문 산출 방식

질문은 아래 형식을 따른다.

```
Q1. [항목명]
- 확인 필요 내용:
- 현재 정보 기준 가정 시 문제점:
- 선택 가능한 옵션(있을 경우):
```

### 질문 예시

- 엑셀 업로드는 신규 등록인가요, 업데이트인가요?
- 업데이트 기준 컬럼은 무엇인가요? (예: 사번, ID 등)
- 실패 리스트에 표시할 대표 식별값은 무엇인가요?
- 각 컬럼의 필수/선택 여부는 어떻게 되나요?
- 특정 컬럼의 유효성 검증 규칙은 무엇인가요?

---

### 질문 완료 시 처리 규칙

- 모든 질문에 대한 답변이 완료되어 정책에 반영되면, 정책서에서 질문 섹션을 삭제한다.
- 질문 섹션이 남아있는 정책서는 미결 사항이 있는 것으로 간주한다.
