---

# 🎯 목적

이 스킬은 사용자가 제공한 "단일 화면 MD 문서"를 다음 두 개의 문서로 분리한다.

1. 📘 화면 정책정의서 (해당 화면 전용 정책)
2. 📗 화면 기능명세서 (구현 상세 명세)

본 스킬은 공통 정책 문서를 생성하지 않는다.  
정책정의서는 반드시 해당 화면에 종속된다.

---

# 🧩 분리 원칙

## 1️⃣ 정책정의서에 포함할 내용

다음 성격의 내용은 정책정의서로 이동한다:

- 해당 화면에서 반드시 지켜야 하는 동작 규칙
- UI 표시 규칙
- 정렬/표기 규칙
- 버튼 노출 조건
- 예외 처리 규칙
- 클릭 동작 원칙
- 상태 표시 기준
- 파일 네이밍 규칙
- 에러 메시지 정책
- 데이터 없음 처리 기준
- 다른 공통 정책과의 충돌 처리 규칙

⚠ 단, 정책은 "해당 화면 범위 내에서만 적용"되도록 작성한다.

예시:
- 본 화면에서는 행 클릭 시 수정 팝업 대신 상세 화면으로 이동한다.
- 본 화면에서는 1월~12월 고정 12행을 유지한다.

---

## 2️⃣ 기능명세서에 포함할 내용

다음 성격의 내용은 기능명세서로 이동한다:

- 화면 ID
- 화면명
- 화면 목적
- 연관 화면
- 데이터 출처
- 합산 대상 필드
- 컬럼 정의
- API 연결 설명
- 데이터 계산 방식
- 상세 화면 이동 경로
- 다운로드 연결 방식

기능명세서는 구현자가 바로 개발 가능한 수준으로 구체적으로 작성한다.

---

# 📁 생성 파일 형식

반드시 두 개의 md 파일을 생성한다.

---

## 📘 파일 1: [화면명]_정책정의서.md

구성:

1. 화면 개요
2. 화면 전용 정책 목록
3. UI 동작 정책
4. 데이터 표시 정책
5. 다운로드/파일 정책
6. 예외 처리 정책
7. 적용 공통 정책 참조 (있을 경우)

---

## 📗 파일 2: [화면명]_기능명세서.md

구성:

1. 화면 기본 정보
2. 데이터 출처
3. 테이블/폼 구성
4. 필드 정의
5. 처리 로직
6. 화면 이동
7. 연결 API 정의 (있을 경우)

---

# 🧠 작성 기준

- 문장형 서술이 아닌 명확한 명세형 문장 사용
- 모호한 표현 금지
- "예상", "아마", "보통" 등의 표현 사용 금지
- 정책과 기능이 섞이지 않도록 엄격히 분리
- 원본 문서의 내용을 누락 없이 재배치

---

# 🚫 금지 사항

- 공통 정책 문서 생성 금지
- 정책을 다른 화면에도 적용되는 형태로 일반화 금지
- 기능 로직을 정책 문서에 포함 금지
- 데이터 출처를 정책 문서에 포함 금지

---

# ✅ 출력 규칙

- 반드시 2개의 md 파일 생성
- 기존 문서 내용을 삭제하지 않고 재구성
- 필요 시 문장 정제 가능하나 의미 변경 금지
