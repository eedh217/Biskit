# 급여대장 관리 화면 - 최종 정책서

---

## 1. 화면 개요

- 모듈명: 급여대장 관리
- 기능: 급여대장 추가 / 수정 / 삭제
- 화면 ID
  - 리스트: HRIS_PR_01
  - 추가 팝업: HRIS_PR_02
  - 수정 팝업: HRIS_PR_03

---

## 2. 리스트 화면 (HRIS_PR_01)

### 2.1 검색영역
- 구성: 인풋박스 + 검색 버튼 `[공통]`
- 검색 대상: 급여대장명 `[화면별]`
- placeholder: `급여대장명을 입력해주세요.` `[공통 - 검색항목 1개 규칙 적용]`
- Enter 입력 시 검색 실행 `[공통 권장]`

### 2.2 리스트 개수(카운트)
- 텍스트로 제공 `[공통]`
- 기본: 등록된 총 리스트 개수 제공 `[공통]`
- 검색 상태: 검색 결과 개수 제공 `[공통]`

### 2.3 리스트(테이블)
- 테이블 컴포넌트 사용 `[공통]`
- 기본 정렬: 최근 등록 순 `[공통]`
- **행 클릭 시 급여내역 상세 화면으로 이동** `[화면별 - 공통 정책 충돌 ①]`
  - 선택 영역: 행 전체 (단, 수정 버튼 영역 제외) `[화면별 - Q6 확정]`
  - 전달 키값: 별도 DB PK(id) `[공통]`
- 가장 좌측에 "행 선택" 체크박스 제공(멀티 선택) `[공통]`
- **가장 우측에 "수정" 버튼 제공** `[화면별 - 공통 정책 충돌 ②]`
  - 수정 버튼 클릭 시 수정 팝업(HRIS_PR_03) 노출
  - **수정 버튼 영역은 행 클릭 대상에서 제외** `[화면별 - Q6 확정]`
  - 전달 키값: 별도 DB PK(id) `[공통]`

#### 테이블 컬럼 `[화면별]`

| 컬럼명 | 타입 | 비고 |
|--------|------|------|
| 급여대장명 | 문자열 | 최대 50자 |
| 귀속연도 | 연도 | YYYY |
| 귀속월 | 월 | M월 |
| 지급일자 | 날짜 | YYYYMMDD |
| 확정여부 | ENUM | 확정 / 미확정 `[화면별 - Q1 확정]` |
| 수정 | 버튼 | 행 우측 수정 버튼 |

### 2.4 리스트 개수 선택(페이지 사이즈)
- 셀렉박스 제공 `[공통]`
- 옵션: 15개 / 30개 / 50개 / 100개 `[공통]`
- 기본 선택: 30개 `[공통]`

### 2.5 엑셀 다운로드
- 버튼 제공 `[공통]`
- 파일명: `HRIS_급여대장_리스트` `[화면별]`
- 다운로드 범위 규칙 `[공통]`
  1. 검색하지 않았을 경우: 전체 리스트 엑셀 다운로드
  2. 검색했을 경우: 검색 결과 리스트 엑셀 다운로드
  3. 리스트 선택 후 버튼 클릭 시: 선택한 리스트만 엑셀 다운로드
- 엑셀 정렬(sorting)은 화면과 동일 `[공통]`

### 2.6 선택 삭제
- 버튼 제공 `[공통]`
- 좌측 체크박스 선택 시 활성화 `[공통]`
- 버튼 클릭 시 confirm 노출 `[공통]`
  - 문구: `총 N개의 리스트를 삭제하시겠습니까?` `[공통]`
- 삭제 후 리스트/카운트 UI는 최신 상태로 갱신 `[공통]`
- **삭제 정책** `[화면별]`
  - 물리삭제
  - 삭제 후 복구 불가
  - **확정된 급여대장은 삭제 불가**
    - 확정 급여대장 삭제 시도 시 에러 문구: `확정된 급여대장은 삭제하실 수 없습니다.`
  - 미확정 급여대장만 삭제 가능
- **삭제 실패 시 처리** `[공통]`
  - 건별 트랜잭션 적용, 부분 성공 허용
  - 삭제 실패 리스트 팝업 제공
  - 실패한 리스트 개수 제공
  - 실패한 리스트 정보와 실패 사유 제공

## 3. 급여대장 추가 팝업 (HRIS_PR_02)

### 3.1 닫기/이탈 방지 (Confirm)
- 팝업 외 영역 클릭 또는 닫기 버튼 클릭 시 confirm 노출 `[공통]`
- ESC, 브라우저 뒤로가기 등은 비활성/차단 `[공통 기본값]`
- confirm 문구: `급여대장 추가를 취소하시겠습니까?` `[공통 - 항목명 치환]`

### 3.2 추가 완료 토스트
- 추가 성공 시 토스트 1.5초 노출 `[공통]`
- 문구: `급여대장 추가를 완료했습니다.` `[공통 - 항목명 치환]`

### 3.3 유효성 검사 (Validation) 공통 규칙
- 검증 타이밍: blur / submit `[공통]`
- 에러 표시 `[공통]`
  - 컴포넌트 하단에 에러 문구 노출
  - 재입력 후 검증 통과 시 에러 문구 제거
  - 포커스 아웃 시 정상 입력으로 판단되면 에러 문구 제거

### 3.4 서버 에러 처리 `[공통]`
- 서버 에러(500 등) 발생 시 에러 alert 노출
- alert 문구: `서버 오류가 발생했습니다. 잠시 후 다시 시도해 주세요.` `[공통]`
- alert 확인 후 팝업 유지 (입력값 보존)

### 3.5 입력 필드 상세 `[화면별]`

> 모든 입력 필드는 필수값입니다.

#### 급여대장명
- 필수
- 단일 인풋박스
- 최대 50자
- 중복 허용 (동일 급여대장명 여러 개 등록 가능) `[화면별 - Q5 확정]`
- 특수문자 불가
  - 검증 시점: blur
  - 에러 문구: `특수문자는 입력하실 수 없습니다.`
- 최대 글자 수 초과 시
  - 검증 시점: blur
  - 에러 문구: `최대 50자까지 입력 가능합니다.`

#### 귀속연도
- 필수
- 단일 선택 셀렉박스
- placeholder: `선택` `[공통]`
- 셀렉박스 옵션: 2024년부터 (현재연도 + 1)년까지 동적 생성 `[화면별]`
  - 예시 (현재 2026년): 2024 / 2025 / 2026 / 2027
  - 연도가 변경되면 셀렉박스 옵션 리스트 자동 갱신

#### 귀속월
- 필수
- 단일 선택 셀렉박스
- placeholder: `선택` `[공통]`
- 셀렉박스 옵션: 1월 / 2월 / 3월 / 4월 / 5월 / 6월 / 7월 / 8월 / 9월 / 10월 / 11월 / 12월

#### 지급일자
- 필수
- Datepicker 컴포넌트 노출 `[화면별]`
- **귀속연월 이후 날짜 검증** `[화면별]`
  - 검증 시점: blur
  - 검증 규칙: 지급일자 ≥ 귀속연도 + 귀속월의 첫째 날 (즉, 지급일자가 귀속연월보다 이전이면 에러)
  - 에러 문구: `지급일자는 귀속연월보다 이후 날짜여야 합니다.`
  - 귀속연도 또는 귀속월이 미선택 상태일 경우: 날짜 비교 검증 생략 `[화면별 - Q2 확정]`

## 4. 급여대장 수정 팝업 (HRIS_PR_03)

### 4.1 확정 상태에 따른 팝업 분기 `[화면별 - Q3 확정]`

수정 버튼 클릭 시 해당 급여대장의 확정여부에 따라 팝업 동작이 분기된다.

| 확정여부 | 팝업 모드 | 필드 상태 | 하단 버튼 |
|----------|-----------|-----------|-----------|
| 미확정 | 수정 모드 | 전체 편집 가능 `[Q4 확정]` | 취소 / 저장 |
| 확정 | 읽기 전용 모드 `[Q3 확정]` | 전체 readonly | 닫기 |

---

### 4.2 미확정 급여대장 - 수정 모드

#### 4.2.1 닫기/이탈 방지 (Confirm)
- 팝업 외 영역 클릭 또는 닫기(취소) 버튼 클릭 시 confirm 노출 `[공통]`
- ESC, 브라우저 뒤로가기 등은 비활성/차단 `[공통 기본값]`
- confirm 문구: `급여대장 수정을 취소하시겠습니까?` `[공통 - 항목명 치환]`

#### 4.2.2 수정 완료 토스트
- 수정 성공 시 토스트 1.5초 노출 `[공통]`
- 문구: `급여대장 수정을 완료했습니다.` `[공통 - 항목명 치환]`

#### 4.2.3 입력 필드 상세
- 추가 팝업과 동일한 입력 필드 및 유효성 규칙 적용
- 모든 필드 수정 가능 `[화면별 - Q4 확정]`
  - 급여대장명 / 귀속연도 / 귀속월 / 지급일자

#### 4.2.4 유효성 검사 (Validation)
- 추가 팝업과 동일 `[공통]`

#### 4.2.5 서버 에러 처리
- 추가 팝업과 동일 (에러 alert 노출 + 팝업 유지) `[공통]`

---

### 4.3 확정 급여대장 - 읽기 전용 모드 `[화면별 - Q3 확정 / 공통 정책 충돌 ③]`

#### 4.3.1 필드 표시
- 모든 입력 필드를 readonly 상태로 표시
  - 급여대장명 / 귀속연도 / 귀속월 / 지급일자
- 값은 저장된 데이터를 그대로 노출
- 편집 불가 (입력, 선택, datepicker 모두 비활성)

#### 4.3.2 하단 버튼
- **'닫기' 버튼만 제공** (취소/저장 버튼 미노출) `[화면별 - Q3 확정]`
- 닫기 버튼 클릭 시 팝업 즉시 닫힘 (confirm 미노출)

#### 4.3.3 팝업 닫기 동작
- 팝업 외 영역 클릭 시 팝업 즉시 닫힘 (confirm 미노출)
- 읽기 전용이므로 데이터 변경이 없어 이탈 방지 불필요

## 5. 상태 변경 규칙 `[화면별]`

- 추가/수정 팝업에서 귀속연도 또는 귀속월 변경 시, 지급일자의 귀속연월 이후 날짜 검증이 재실행됨
  - 이미 입력된 지급일자가 변경된 귀속연월보다 이전이 되면 에러 노출
- 귀속연도 또는 귀속월이 미선택 상태이면 지급일자 날짜 비교 검증 생략 `[Q2 확정]`

## 6. 충돌 지점 정리

| # | 항목 | 공통 정책 | 화면별 정책 | 적용 결과 |
|---|------|-----------|-------------|-----------|
| ① | 행 클릭 시 동작 | 수정 팝업 노출 | 급여내역 상세 화면으로 이동 | **화면별 정책 우선 적용** - 행 클릭 시 급여내역 상세 화면으로 이동 |
| ② | 수정 기능 진입점 | 행 클릭 시 수정 팝업 | 행 우측 수정 버튼 클릭 시 수정 팝업 | **화면별 정책 우선 적용** - 행 우측 수정 버튼으로 수정 팝업 진입 |
| ③ | 확정 급여대장 수정 팝업 | 수정 팝업: 편집 가능 + 취소/저장 버튼 + confirm | 확정 급여대장: 읽기 전용 + 닫기 버튼만 + confirm 미노출 | **화면별 정책 우선 적용** - 확정 급여대장은 읽기 전용 모드로 표시 |

> - 충돌 ①②: 공통 정책의 "행 클릭 → 수정 팝업"이 본 화면에서는 행 클릭(상세 이동) / 수정 버튼(수정 팝업)으로 분리됩니다.
> - 충돌 ③: 공통 정책의 수정 팝업은 항상 편집 가능하지만, 확정 급여대장은 읽기 전용 + 닫기 버튼만 제공하여 이탈 방지 confirm도 생략됩니다.
