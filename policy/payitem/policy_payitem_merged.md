# 급여항목 관리 화면 - 최종 정책서

---

## 1. 화면 개요

- 모듈명: 급여항목 관리
- 기능: 급여항목 추가 / 수정 / 삭제
- 화면 ID
  - 리스트: HRIS_PI_01
  - 추가 팝업: HRIS_PI_02
  - 수정 팝업: HRIS_PI_03

---

## 2. 리스트 화면 (HRIS_PI_01)

### 2.1 검색영역
- 구성: 인풋박스 + 검색 버튼 `[공통]`
- 검색 대상: 급여코드 OR 항목명 `[화면별]`
- placeholder: `급여코드 또는 항목명을 입력해주세요.` `[공통]`
- Enter 입력 시 검색 실행 `[공통 권장]`

### 2.2 리스트 개수(카운트)
- 텍스트로 제공 `[공통]`
- 기본: 등록된 총 리스트 개수 제공 `[공통]`
- 검색 상태: 검색 결과 개수 제공 `[공통]`

### 2.3 리스트(테이블)
- 테이블 컴포넌트 사용 `[공통]`
- 기본 정렬: 최근 등록 순 `[공통]`
- 행 클릭 시 수정 팝업(HRIS_PI_03) 노출 `[공통]`
  - 선택 영역: 행 전체 `[공통]`
  - 전달 키값: 별도 DB PK(id) `[공통]`
- 가장 좌측에 "행 선택" 체크박스 제공(멀티 선택) `[공통]`

#### 테이블 컬럼 `[화면별]`

| 컬럼명 | 타입 | 비고 |
|--------|------|------|
| 구분 | ENUM | 지급항목 / 공제항목 |
| 급여코드 | 문자열 | 최대 10자 |
| 항목명 | 문자열 | 최대 30자 |
| 유형 | 문자열 | 구분에 따라 값 상이 |

### 2.4 리스트 개수 선택(페이지 사이즈)
- 셀렉박스 제공 `[공통]`
- 옵션: 15개 / 30개 / 50개 / 100개 `[공통]`
- 기본 선택: 30개 `[공통]`

### 2.5 엑셀 다운로드
- 버튼 제공 `[공통]`
- 파일명: `HRIS_급여항목_리스트` `[화면별]`
- 다운로드 범위 규칙 `[공통]`
  1. 검색하지 않았을 경우: 전체 리스트 엑셀 다운로드
  2. 검색했을 경우: 검색 결과 리스트 엑셀 다운로드
  3. 리스트 선택 후 버튼 클릭 시: 선택한 리스트만 엑셀 다운로드
- 엑셀 정렬(sorting)은 화면과 동일 `[공통]`

### 2.6 선택 삭제
- 버튼 제공 `[공통]`
- 좌측 체크박스 선택 시 활성화 `[공통]`
- 버튼 클릭 시 confirm 노출 `[공통]`
  - 문구: `총 N개의 리스트를 삭제하시겠습니까?` `[공통]`
- 삭제 후 리스트/카운트 UI는 최신 상태로 갱신 `[공통]`
- **삭제 연관 데이터 안내 confirm** `[화면별 - Q3 확정]`
  - 삭제 버튼 클릭 시 공통 confirm 이후, 추가 안내 confirm 노출
  - 문구: `급여항목을 삭제하시겠습니까? 급여항목 삭제 시, 급여대장에 저장된 데이터가 모두 삭제됩니다.`
- **삭제 정책** `[화면별 - Q3 확정]`
  - 물리삭제
  - 삭제 후 복구 불가
  - **확정된 급여대장에 사용된 급여항목은 삭제 불가**
    - 삭제 시도 시 에러 처리 (삭제 실패 사유: 확정 급여대장에서 사용 중)
  - 미확정 급여대장에 사용된 급여항목은 삭제 가능
    - 삭제 시 해당 급여대장의 연관 데이터도 함께 삭제
- **삭제 실패 시 처리** `[공통]`
  - 건별 트랜잭션 적용, 부분 성공 허용
  - 삭제 실패 리스트 팝업 제공
  - 실패한 리스트 개수 제공
  - 실패한 리스트 정보와 실패 사유 제공

## 3. 급여항목 추가 팝업 (HRIS_PI_02)

### 3.1 닫기/이탈 방지 (Confirm)
- 팝업 외 영역 클릭 또는 닫기 버튼 클릭 시 confirm 노출 `[공통]`
- ESC, 브라우저 뒤로가기 등은 비활성/차단 `[공통 기본값]`
- confirm 문구: `급여항목 추가를 취소하시겠습니까?` `[공통 - 항목명 치환]`

### 3.2 추가 완료 토스트
- 추가 성공 시 토스트 1.5초 노출 `[공통]`
- 문구: `급여항목 추가를 완료했습니다.` `[공통 - 항목명 치환]`

### 3.3 유효성 검사 (Validation) 공통 규칙
- 검증 타이밍: blur / submit `[공통]`
- 에러 표시 `[공통]`
  - 컴포넌트 하단에 에러 문구 노출
  - 재입력 후 검증 통과 시 에러 문구 제거
  - 포커스 아웃 시 정상 입력으로 판단되면 에러 문구 제거

### 3.4 서버 에러 처리 `[공통]`
- 서버 에러(500 등) 발생 시 에러 alert 노출
- alert 문구: `서버 오류가 발생했습니다. 잠시 후 다시 시도해 주세요.` `[공통]`
- alert 확인 후 팝업 유지 (입력값 보존)

### 3.5 입력 필드 상세 `[화면별]`

#### 구분
- 필수
- 라디오 버튼: 지급항목 / 공제항목
- 기본 선택값: 지급항목 `[화면별 - Q4 확정]`
- 구분 변경 시 유형 셀렉박스 초기화 (`선택` 상태로 복원) `[화면별]`

#### 급여코드
- 필수
- 단일 인풋박스
- 최대 10자
- 특수문자 불가
  - 에러 문구: `특수문자는 입력하실 수 없습니다.`
- 최대 글자 수 초과 시
  - 에러 문구: `최대 10자까지 입력 가능합니다.`
- 중복 불가 (급여코드 기준)
  - 검증 시점: submit (저장 버튼 클릭 시) `[화면별]`
  - 에러 문구: `중복된 급여코드는 사용하실 수 없습니다.`

#### 항목명
- 필수
- 단일 인풋박스
- 최대 30자
- 특수문자 불가
  - 에러 문구: `특수문자는 입력하실 수 없습니다.`
- 최대 글자 수 초과 시
  - 에러 문구: `최대 30자까지 입력 가능합니다.`

#### 유형
- 필수
- 단일 선택 셀렉박스
- placeholder: `선택` `[공통]`
- **구분(지급항목/공제항목) 선택에 따라 셀렉박스 옵션이 분기됨** `[화면별]`

| 구분 선택값 | 셀렉박스 옵션 |
|-------------|---------------|
| 지급항목 | 자가운전보조금 / 비과세 식대 / 취재수당 / 벽지수당 / 이주수당 / 재해급여 |
| 공제항목 | 국민연금 / 건강보험 / 고용보험 / 소득세 / 지방소득세 |

- 구분 미선택 시 유형 셀렉박스 비활성화 (disabled) `[화면별]`
- 구분 변경 시 기존 선택된 유형 값 초기화 `[화면별]`

## 4. 급여항목 수정 팝업 (HRIS_PI_03)

### 4.1 기본 동작
- 기능/검증/토스트/confirm 정책은 "추가 팝업"과 동일 `[공통]`

### 4.2 닫기/이탈 방지 (Confirm)
- 팝업 외 영역 클릭 또는 닫기 버튼 클릭 시 confirm 노출 `[공통]`
- confirm 문구: `급여항목 수정을 취소하시겠습니까?` `[공통 - 항목명 치환]`

### 4.3 수정 완료 토스트
- 수정 성공 시 토스트 1.5초 노출 `[공통]`
- 문구: `급여항목 수정을 완료했습니다.` `[공통 - 항목명 치환]`

### 4.4 수정 제한 `[화면별 - Q1, Q2 확정]`
- 구분: 수정 불가 (읽기 전용) `[화면별 - Q2 확정]`
- 급여코드: 수정 가능 `[화면별 - Q1 확정]`
- 항목명: 수정 가능
- 유형: 수정 가능 (구분이 읽기 전용이므로 동일 구분 내 옵션만 표시)

### 4.5 급여코드 중복 검사 (수정 시) `[화면별 - Q1 확정]`
- 자기 자신을 포함하여 중복 검사
- 급여코드를 변경하지 않으면 중복 검사 통과

### 4.6 서버 에러 처리 `[공통]`
- 추가 팝업과 동일 (에러 alert 노출 + 팝업 유지)

## 5. 상태 변경 규칙 `[화면별]`

- 추가 팝업에서 구분(지급항목 ↔ 공제항목) 변경 시 유형 셀렉박스 초기화
  - 선택된 유형 값이 해제되고 `선택` placeholder로 복원
  - 변경된 구분에 맞는 옵션 리스트로 교체
- 수정 팝업에서는 구분이 읽기 전용이므로 유형 초기화 불발생 `[Q2 확정]`

## 6. 충돌 지점 정리

| 항목 | 공통 정책 | 화면별 정책 | 적용 결과 |
|------|-----------|-------------|-----------|
| 충돌 없음 | - | - | 공통 정책과 화면별 정책 간 충돌 지점 없음 |

> 본 화면은 공통 정책과 화면별 정책 간 충돌이 발견되지 않았습니다. 화면별 정책은 공통 정책을 보완하는 형태로 적용됩니다.
