# 임직원 관리 화면 - 최종 정책서

---

## 1. 화면 개요

- 모듈명: 임직원 관리
- 기능: 임직원 추가 / 수정 / 삭제
- 화면 ID
  - 리스트: HRIS_EP_01
  - 추가 팝업: HRIS_EP_02
  - 수정 팝업: HRIS_EP_03

---

## 2. 리스트 화면 (HRIS_EP_01)

### 2.1 검색영역
- 구성: 인풋박스 + 검색 버튼 `[공통]`
- 검색 대상: 사번 OR 성명 `[화면별]`
- placeholder: `사번 또는 성명을 입력해주세요.` `[화면별]`
- Enter 입력 시 검색 실행 `[공통 권장]`

### 2.2 리스트 개수(카운트)
- 텍스트로 제공 `[공통]`
- 기본: 등록된 총 리스트 개수 제공 `[공통]`
- 검색 상태: 검색 결과 개수 제공 `[공통]`

### 2.3 리스트(테이블)
- 테이블 컴포넌트 사용 `[공통]`
- 기본 정렬: 최근 등록 순 `[공통]`
- 행 클릭 시 수정 팝업(HRIS_EP_03) 노출 `[공통]`
  - 선택 영역: 행 전체 `[공통]`
  - **전달 키값: 별도 DB PK(id)** `[화면별 - Q1 확정]`
- 가장 좌측에 "행 선택" 체크박스 제공(멀티 선택) `[공통]`

#### 테이블 컬럼 `[화면별]`

| 컬럼명 | 타입 | 비고 |
|--------|------|------|
| 사번 | 문자열 | |
| 성명 | 문자열 | |
| 내외국인 여부 | ENUM | 내국인 / 외국인 |
| 이메일 | 문자열 | |
| 입사일 | YYYYMMDD | |
| 퇴사일 | YYYYMMDD / NULL | NULL일 경우 빈 값 표시 |
| 연락처 | 문자열 | |
| 장애여부 | ENUM | 비장애인 / 장애인복지법상 장애인 / 국가유공자 중증환자 / 중증환자 |

### 2.4 리스트 개수 선택(페이지 사이즈)
- 셀렉박스 제공 `[공통]`
- 옵션: 15개 / 30개 / 50개 / 100개 `[공통]`
- 기본 선택: 30개 `[공통]`

### 2.5 엑셀 다운로드
- 버튼 제공 `[공통]`
- 파일명: `HIRS_유저_리스트` `[화면별]`
- 다운로드 범위 규칙 `[공통]`
  1. 검색하지 않았을 경우: 전체 리스트 엑셀 다운로드
  2. 검색했을 경우: 검색 결과 리스트 엑셀 다운로드
  3. 리스트 선택 후 버튼 클릭 시: 선택한 리스트만 엑셀 다운로드
- 엑셀 정렬(sorting)은 화면과 동일 `[공통]`

### 2.6 선택 삭제
- 버튼 제공 `[공통]`
- 좌측 체크박스 선택 시 활성화 `[공통]`
- 버튼 클릭 시 confirm 노출 `[공통]`
  - 문구: `총 N개의 리스트를 삭제하시겠습니까?` `[공통]`
- 삭제 후 리스트/카운트 UI는 최신 상태로 갱신 `[공통]`
- **삭제 정책** `[화면별]`
  - 물리삭제 `[보완]`
  - 급여/연말정산 데이터 함께 삭제 `[보완]`
  - 삭제 후 복구 불가 `[보완]`
  - 삭제된 사번은 재사용 가능
- **삭제 실패 시 처리** `[공통 - Q2 확정]`
  - 건별 트랜잭션 적용, 부분 성공 허용
  - 삭제 실패 리스트 팝업 제공
  - 실패한 리스트 개수 제공
  - 실패한 리스트 정보와 실패 사유 제공

## 3. 임직원 추가 팝업 (HRIS_EP_02)

### 3.1 닫기/이탈 방지 (Confirm)
- 팝업 외 영역 클릭 또는 닫기 버튼 클릭 시 confirm 노출 `[공통]`
- ESC, 브라우저 뒤로가기 등은 비활성/차단 `[공통 기본값]`
- confirm 문구: `임직원 추가를 취소하시겠습니까?` `[공통 - 항목명 치환]`

### 3.2 추가 완료 토스트
- 추가 성공 시 토스트 1.5초 노출 `[공통]`
- 문구: `임직원 추가를 완료했습니다.` `[공통 - 항목명 치환]`

### 3.3 유효성 검사 (Validation) 공통 규칙
- 검증 타이밍: blur / submit `[공통]`
- 에러 표시 `[공통]`
  - 컴포넌트 하단에 에러 문구 노출
  - 재입력 후 검증 통과 시 에러 문구 제거
  - 포커스 아웃 시 정상 입력으로 판단되면 에러 문구 제거

### 3.4 서버 에러 처리 `[공통]`
- 서버 에러(500 등) 발생 시 에러 alert 노출
- alert 문구: `서버 오류가 발생했습니다. 잠시 후 다시 시도해 주세요.` `[공통]`
- alert 확인 후 팝업 유지 (입력값 보존)

### 3.5 입력 필드 상세 `[화면별 + 보완]`

#### 사번
- 필수
- 최대 30자
- 허용 특수문자 32개: ``!"#$%&'()*+,-./:;<>=?@[₩]^_`{|}~`` 
- 중복 불가
  - 검증 시점: submit 
  - 에러 문구: `중복된 사번은 사용하실 수 없습니다.` 

#### 성명
- 필수
- 최대 30자
- 특수문자 불가
  - 에러 문구: `특수문자는 입력하실 수 없습니다.` 

#### 내외국인 여부 
- 필수
- ENUM: 내국인 / 외국인

---

#### [내국인 분기]

**주민등록번호:**
- 필수 (내국인 선택 시)
- 인풋박스 2개 (앞 6자리 - 뒤 7자리) `[보완]`
- 숫자만 입력 가능 `[보완]`
- 마스킹 처리 없이 전체 표시 `[보완]`
- blur 시 검증:
  - 앞 6자리 날짜 유효성
  - 뒤 7자리 첫 숫자: 1900년대(1,2) / 2000년대(3,4)
  - 에러 문구: `유효하지 않은 주민등록번호입니다.`

---

#### [외국인 분기]

**외국인등록번호:**
- 외국인등록번호 / 여권번호 중 하나 필수
- 인풋박스 2개 (앞 6자리 - 뒤 7자리) `[보완]`
- 숫자만 입력 가능 `[보완]`
- 마스킹 없음 `[보완]`
- blur 시 검증:
  - 앞 6자리 날짜 유효성
  - 뒤 7자리 첫 숫자: 1900년대(5,6) / 2000년대(7,8)
  - 에러 문구: `유효하지 않은 외국인등록번호입니다.`

**여권번호:** `[보완]`
- 단일 인풋박스
- 특수문자 입력 불가
- 여권 선택 시 추가 필수 입력:
  - 생년월일: YYYYMMDD, 숫자 8자리
  - 성별: 라디오 버튼 (남 / 여)

**국적:**
- 필수 (외국인 선택 시)
- 전세계 국가 리스트 제공 `[보완]`
- 가나다순 정렬 `[보완]`
- 고정 리스트 사용 `[보완]`

---

#### 거주구분 `[보완]`
- 필수
- 라디오 버튼 (거주자 / 비거주자)

#### 장애여부 
- 필수
- ENUM 4개 옵션:
  - 비장애인
  - 장애인복지법상 장애인
  - 국가유공자 중증환자
  - 중증환자

#### 이메일
- 필수
- blur: 형식 검사
  - 에러 문구: `이메일 형식이 맞지 않습니다.`
- submit: 퇴사일 NULL인 임직원과 중복 불가
  - 에러 문구: `근로자 중 중복된 이메일이 등록되어 있습니다.`

#### 연락처 
- 필수
- 허용 문자: + ( ) - 숫자
- 최대 30자

#### 휴대폰번호 `[보완]`
- 필수
- 인풋박스 3개 (3자리 - 3자리 - 4자리)
- 숫자만 입력 가능

#### 입사일
- 필수
- YYYYMMDD
- blur 시 유효성 검사
- 에러 문구: `유효하지 않은 날짜입니다.`

#### 퇴사일
- YYYYMMDD
- NULL 허용 (선택 입력)
- blur 시 유효성 검사
- 에러 문구: `유효하지 않은 날짜입니다.`

#### 주소
- 필수
- 외부 무료 주소 검색 API 사용
- **주소 검색 팝업** `[보완]`
  - 검색 인풋 1개 + 검색 버튼
  - 안내 문구: `도로명 또는 지번으로 검색하세요.`
  - 검색 결과 리스트 표시 항목:
    - 우편번호
    - 도로명주소 또는 지번주소
  - 결과 클릭 시:
    - 팝업 닫힘
    - 우편번호/주소 자동 입력
    - 상세주소 입력창으로 포커스 이동

## 4. 임직원 수정 팝업 (HRIS_EP_03)

### 4.1 기본 동작
- 기능/검증/토스트/confirm 정책은 "추가 팝업"과 동일 `[공통]`

### 4.2 닫기/이탈 방지 (Confirm)
- 팝업 외 영역 클릭 또는 닫기 버튼 클릭 시 confirm 노출 `[공통]`
- confirm 문구: `임직원 수정을 취소하시겠습니까?` `[공통 - 항목명 치환]`

### 4.3 수정 완료 토스트
- 수정 성공 시 토스트 1.5초 노출 `[공통]`
- 문구: `임직원 수정을 완료했습니다.` `[공통 - 항목명 치환]`

### 4.4 수정 제한 `[화면별]`
- 사번 수정 불가 (읽기 전용) `[보완]`
- 나머지 입력 필드 정책은 추가 팝업과 동일

### 4.5 이메일 중복 검사 (수정 시) `[Q15 확정]`
- 자기 자신을 포함하여 중복 검사
- 이메일을 변경하지 않으면 중복 검사 통과

### 4.6 서버 에러 처리 `[공통 - Q16 확정]`
- 추가 팝업과 동일 (에러 alert 노출 + 팝업 유지)
