# 급여내역 상세 화면 - 최종 정책서

---

## 1. 화면 개요

- 모듈명: 급여내역 상세
- 진입 경로: 급여대장 관리 리스트(HRIS_PR_01)에서 행 클릭 시 이동
- 기능: 급여내역 임직원 추가 / 급여항목 관리(토글) / 급여내역 수정 / 급여확정·해제 / 선택 삭제
- 화면 ID
  - 급여내역 상세 리스트: HRIS_PD_01
  - 임직원 추가 팝업: HRIS_PD_02
  - 급여항목 관리 팝업: HRIS_PD_03
  - 급여내역 수정 팝업: HRIS_PD_04

## 2. 급여내역 상세 리스트 화면 (HRIS_PD_01)

### 2.1 검색영역
- 구성: 인풋박스 + 검색 버튼 `[공통]`
- 검색 대상: 사번 OR 성명 `[화면별]`
- placeholder: `사번 또는 성명을 입력해주세요.` `[공통 - 검색항목 2개 규칙 적용]`
- Enter 입력 시 검색 실행 `[공통 권장]`

### 2.2 리스트 개수(카운트)
- 텍스트로 제공 `[공통]`
- 기본: 등록된 총 리스트 개수 제공 `[공통]`
- 검색 상태: 검색 결과 개수 제공 `[공통]`

### 2.3 리스트(테이블)
- 테이블 컴포넌트 사용 `[공통]`
- 기본 정렬: 최근 등록 순 `[공통]`
- 행 클릭 시 급여내역 수정 팝업(HRIS_PD_04) 노출 `[공통]`
  - 선택 영역: 행 전체 `[공통]`
  - 전달 키값: 별도 DB PK(id) `[공통]`
- 가장 좌측에 "행 선택" 체크박스 제공(멀티 선택) `[공통]`

#### 테이블 컬럼 `[화면별]`

테이블 컬럼은 **고정 컬럼 + 동적 컬럼(급여항목 토글에 따라 변동)** 으로 구성된다.

| 구분 | 컬럼명 | 타입 | 비고 |
|------|--------|------|------|
| 고정 | 체크박스 | 체크박스 | 좌측 행 선택 체크박스 `[공통]` |
| 고정 | 사번 | 문자열 | |
| 고정 | 성명 | 문자열 | |
| 동적 (지급항목 그룹) | [급여항목 관리에서 토글 ON된 지급항목들] | 금액 | 급여항목 관리 팝업에서 ON 설정된 지급항목만 노출 |
| 고정 | 지급항목 합계 | 금액 | 지급항목 그룹 가장 우측. 해당 행의 지급항목 합산 |
| 동적 (공제항목 그룹) | [급여항목 관리에서 토글 ON된 공제항목들] | 금액 | 급여항목 관리 팝업에서 ON 설정된 공제항목만 노출 |
| 고정 | 공제항목 합계 | 금액 | 공제항목 그룹 가장 우측. 해당 행의 공제항목 합산 |

- **동적 컬럼 배치 규칙**: 급여항목 토글을 ON으로 설정하면 해당 그룹(지급항목/공제항목) 내 가장 오른쪽(합계 컬럼 바로 왼쪽)에 추가된다 `[화면별]`
- **합계 컬럼**: 각 그룹의 동적 컬럼 값을 합산하여 표시한다 `[화면별]`
- **금액 표시**: 천 단위 콤마(,) 적용 `[공통 - 금액 정책]`

### 2.4 버튼 영역 `[화면별]`

| 버튼명 | 기능 | 비고 |
|--------|------|------|
| 임직원 추가 | 임직원 추가 팝업(HRIS_PD_02) 노출 | |
| 급여항목 관리 | 급여항목 관리 팝업(HRIS_PD_03) 노출 | |
| 급여확정 / 급여확정해제 | 급여 확정 상태 토글 | 상태에 따라 버튼 라벨 변경 |
| 선택 삭제 | 선택된 임직원 급여내역 삭제 | 좌측 체크박스 선택 시 활성화 `[공통]` |

### 2.5 리스트 개수 선택(페이지 사이즈)
- 셀렉박스 제공 `[공통]`
- 옵션: 15개 / 30개 / 50개 / 100개 `[공통]`

### 2.6 엑셀 다운로드
- 버튼 제공 `[공통]`
- 파일명: `HRIS_급여내역_리스트` `[화면별]`
- 다운로드 범위 규칙 `[공통]`
  1. 검색하지 않았을 경우: 전체 리스트 엑셀 다운로드
  2. 검색했을 경우: 검색 결과 리스트 엑셀 다운로드
  3. 리스트 선택 후 버튼 클릭 시: 선택한 리스트만 엑셀 다운로드
- 엑셀 정렬(sorting)은 화면과 동일 `[공통]`
- 엑셀 컬럼은 화면에 표시된 동적 컬럼(ON 상태)과 동일하게 반영 `[화면별]`
- **합계 컬럼(지급항목 합계/공제항목 합계) 포함 - 값으로 제공** `[화면별 - Q5 확정]`

### 2.7 선택 삭제
- 버튼 제공 `[공통]`
- 좌측 체크박스 선택 시 활성화 `[공통]`
- 버튼 클릭 시 confirm 노출 `[공통]`
  - 문구: `총 N개의 리스트를 삭제하시겠습니까?` `[공통]`
- 삭제 후 리스트/카운트 UI는 최신 상태로 갱신 `[공통]`
- **삭제 정책** `[화면별]`
  - 물리삭제 (해당 임직원의 급여내역 데이터 삭제)
  - 삭제 후 복구 불가
- **삭제 실패 시 처리** `[공통]`
  - 건별 트랜잭션 적용, 부분 성공 허용
  - 삭제 실패 리스트 팝업 제공
  - 실패한 리스트 개수 제공
  - 실패한 리스트 정보와 실패 사유 제공

## 3. 급여확정 / 급여확정해제 `[화면별]`

### 3.1 급여확정

- **급여확정 버튼** 클릭 시 confirm 노출
- confirm 문구:
  ```
  급여확정하시겠습니까?
  급여확정 시, 급여내역 추가/수정/삭제가 제한됩니다.
  ※ 급여확정 해제 후 수정 가능합니다.
  ```
- 확인 버튼 클릭 시:
  - 해당 급여대장의 확정여부가 **확정** 상태로 변경
  - **급여확정** 버튼이 **급여확정해제** 버튼으로 변경
- **확정 상태에서 비활성화되는 요소:**
  - 리스트 좌측 체크박스 (전체 비활성화)
  - 선택 삭제 버튼
  - 임직원 추가 버튼
  - 급여항목 관리 버튼
- **확정 상태에서 급여내역 수정 팝업 동작:** → §5.3 참조

### 3.2 급여확정해제

- **급여확정해제 버튼** 클릭 시 confirm 노출
- confirm 문구:
  ```
  급여확정을 해제하시겠습니까?
  급여확정 해제 시, 급여내역 추가/수정/삭제가 가능합니다.
  ```
- 확인 버튼 클릭 시:
  - 해당 급여대장의 확정여부가 **미확정** 상태로 변경
  - **급여확정해제** 버튼이 **급여확정** 버튼으로 변경
  - 비활성화되었던 요소 모두 정상 활성화 복원

## 4. 임직원 추가 팝업 (HRIS_PD_02)

### 4.1 팝업 개요
- 급여내역에 임직원을 추가하기 위한 선택 팝업
- 임직원 리스트를 제공하고, 선택하여 추가할 수 있다

### 4.2 검색영역 `[화면별 - Q1 확정]`
- 구성: 인풋박스 + 검색 버튼
- 검색 대상: 사번 OR 성명
- placeholder: `사번 또는 성명을 입력해주세요.`
- Enter 입력 시 검색 실행 `[공통 권장]`

### 4.3 임직원 리스트 제공 조건 `[화면별]`
- **'임직원 관리'에 등록된 임직원** 중 아래 조건을 **모두** 충족하는 임직원만 제공:
  1. 해당 급여대장의 **귀속연월에 근무한 날짜가 1일 이상**인 임직원
     - 판정 기준: 임직원의 **입사일 ~ 퇴사일** 구간이 귀속연월과 1일 이상 겹치는 경우
     - 퇴사일이 NULL(재직 중)인 경우: 입사일이 귀속연월 말일 이하이면 해당
     - 예시) 귀속연월 2026-02: 입사일 ≤ 2026-02-28 AND (퇴사일 ≥ 2026-02-01 OR 퇴사일 IS NULL)
  2. 해당 급여대장에 **아직 추가되지 않은** 임직원

### 4.4 리스트 표시 정보
- 사번
- 성명
- 입사일
- 퇴사일

### 4.5 선택 및 추가
- 체크박스를 통한 **다중 선택** 가능 `[화면별]`
- 추가 버튼 클릭 시 선택된 임직원을 급여내역에 추가
- 추가 완료 시 토스트 1.5초 노출 `[공통]`
  - 문구: `임직원 추가를 완료했습니다.` `[공통 - 항목명 치환]`
- 추가 완료 후 팝업 닫힘 및 리스트/카운트 갱신

### 4.6 닫기/이탈 방지 (Confirm)
- 팝업 외 영역 클릭 또는 닫기 버튼 클릭 시 confirm 노출 `[공통]`
- ESC, 브라우저 뒤로가기 등은 비활성/차단 `[공통 기본값]`
- confirm 문구: `임직원 추가를 취소하시겠습니까?` `[공통 - 항목명 치환]`

### 4.7 서버 에러 처리 `[공통]`
- 서버 에러(500 등) 발생 시 에러 alert 노출
- alert 문구: `서버 오류가 발생했습니다. 잠시 후 다시 시도해 주세요.` `[공통]`
- alert 확인 후 팝업 유지

## 5. 급여내역 수정 팝업 (HRIS_PD_04)

### 5.1 기본 동작
- 리스트에서 행 클릭 시 노출 `[공통]`
- 전달 키값: 별도 DB PK(id) `[공통]`

### 5.2 미확정 상태 - 수정 모드

#### 5.2.1 필드 표시

| 필드명 | 상태 | 타입 | 비고 |
|--------|------|------|------|
| 사번 | 읽기 전용 | 문자열 | 수정 불가 `[화면별]` |
| 성명 | 읽기 전용 | 문자열 | 수정 불가 `[화면별]` |
| [ON 상태인 지급항목들] | 편집 가능 | 금액 | 숫자만 입력 가능, 선택 입력(빈 값 허용) |
| [ON 상태인 공제항목들] | 편집 가능 | 금액 | 숫자만 입력 가능, 선택 입력(빈 값 허용) |

- 급여항목 관리 팝업에서 ON으로 설정된 급여항목만 입력 필드로 제공 `[화면별]`
- **숫자만 입력 가능** `[화면별]`
- **빈 값(미입력) 상태로 저장 가능** `[화면별]`
- **미입력 시 NULL 저장** `[화면별 - Q2 확정]`
  - 리스트/엑셀에서 빈 값으로 표시
  - 합계 계산 시 NULL은 제외 (0으로 취급하지 않음)
- 금액 입력 시 천 단위 콤마(,) 적용 `[공통 - 금액 정책]`
- 0 이상의 정수만 허용 `[공통 - 금액 정책]`
- 소수점 불가 `[공통 - 금액 정책]`
- 음수 불가 `[공통 - 금액 정책]`

#### 5.2.2 닫기/이탈 방지 (Confirm)
- 팝업 외 영역 클릭 또는 닫기(취소) 버튼 클릭 시 confirm 노출 `[공통]`
- ESC, 브라우저 뒤로가기 등은 비활성/차단 `[공통 기본값]`
- confirm 문구: `급여내역 수정을 취소하시겠습니까?` `[공통 - 항목명 치환]`

#### 5.2.3 수정 완료 토스트
- 수정 성공 시 토스트 1.5초 노출 `[공통]`
- 문구: `급여내역 수정을 완료했습니다.` `[공통 - 항목명 치환]`

#### 5.2.4 하단 버튼
- 취소 / 수정 버튼 제공

#### 5.2.5 유효성 검사 (Validation)
- 검증 타이밍: blur / submit `[공통]`
- 에러 표시 `[공통]`
  - 컴포넌트 하단에 에러 문구 노출
  - 재입력 후 검증 통과 시 에러 문구 제거
  - 포커스 아웃 시 정상 입력으로 판단되면 에러 문구 제거

#### 5.2.6 서버 에러 처리 `[공통]`
- 서버 에러(500 등) 발생 시 에러 alert 노출
- alert 문구: `서버 오류가 발생했습니다. 잠시 후 다시 시도해 주세요.` `[공통]`
- alert 확인 후 팝업 유지 (입력값 보존)

### 5.3 확정 상태 - 읽기 전용 모드 `[화면별 - 공통 정책 충돌 ①]`

#### 5.3.1 필드 표시
- 모든 필드를 readonly 상태로 표시
  - 사번 / 성명 / 모든 급여항목 금액
- 값은 저장된 데이터를 그대로 노출
- 편집 불가

#### 5.3.2 하단 버튼
- **'닫기' 버튼만 제공** (취소/수정 버튼 미노출) `[화면별]`
- 닫기 버튼 클릭 시 팝업 즉시 닫힘 (confirm 미노출)

#### 5.3.3 팝업 닫기 동작
- 팝업 외 영역 클릭 시 팝업 즉시 닫힘 (confirm 미노출)
- 읽기 전용이므로 데이터 변경이 없어 이탈 방지 불필요

## 6. 급여항목 관리 팝업 (HRIS_PD_03)

### 6.1 팝업 개요
- 급여내역 리스트에 표시할 급여항목을 관리하는 팝업
- **지급항목 / 공제항목** 탭으로 분리 `[화면별]`

### 6.2 탭 구성

| 탭명 | 내용 |
|------|------|
| 지급항목 | 급여항목 관리(HRIS_PI)에서 구분이 '지급항목'인 항목 리스트 |
| 공제항목 | 급여항목 관리(HRIS_PI)에서 구분이 '공제항목'인 항목 리스트 |

### 6.3 리스트 정보

| 컬럼명 | 타입 | 비고 |
|--------|------|------|
| 급여코드 | 문자열 | 급여항목 관리에서 등록된 급여코드 |
| 항목명 | 문자열 | 급여항목 관리에서 등록된 항목명 |
| 사용 여부 | 토글 버튼 | ON: 급여내역 리스트에 컬럼 노출 / OFF: 미노출 |

- 정렬: **등록 오래된 순** (급여항목 관리에서의 등록 순서 기준) `[화면별 - 공통 정책 충돌 ②]`

### 6.4 토글 ON 동작
- 토글을 ON으로 변경 (팝업 내 시각적 변경, 저장 전까지 서버 미반영)
- **저장 시**: 급여내역 리스트(HRIS_PD_01)에 해당 급여항목 컬럼이 추가됨
- **추가 위치**: 해당 그룹(지급항목/공제항목) 내 가장 오른쪽 (합계 컬럼 바로 왼쪽)

### 6.5 토글 OFF 동작
- 토글을 OFF로 변경 시 **alert 노출** `[화면별]`
- alert 문구:
  ```
  사용 -> 미사용으로 변경 시, 입력된 내용은 모두 삭제되며, 데이터는 복구되지 않습니다.
  ```
- alert 확인 버튼 클릭 시:
  - 토글이 OFF 상태로 변경 (팝업 내 시각적 변경)
  - **저장 시**: 급여내역 리스트에서 해당 급여항목 컬럼 삭제 및 입력된 모든 급여내역 데이터 삭제 (복구 불가)
- alert 취소 버튼 클릭 시:
  - 토글 상태 유지 (ON 상태 복원)
  - 변경 없음

### 6.6 저장 `[화면별 - Q3 확정]`
- **저장 버튼 제공** (일괄 반영 방식)
- 저장 버튼 클릭 시 팝업 내 모든 토글 변경사항을 서버에 일괄 반영
- 저장 완료 시:
  - 급여내역 리스트(HRIS_PD_01)에 변경사항 반영 (컬럼 추가/삭제, 데이터 삭제)
  - 토스트 1.5초 노출
  - 문구: `급여항목 설정을 완료했습니다.`
  - 팝업 닫힘

### 6.7 팝업 닫기 `[화면별 - Q4 확정]`
- 닫기 버튼 클릭 시 팝업 닫힘
- **닫기 시 confirm 미노출** (토글 OFF 변경 시 이미 alert으로 안내가 제공되므로 별도 confirm 불필요)
- 저장하지 않고 닫을 경우 변경사항은 반영되지 않음

## 7. 귀속연월 변경 시 임직원 검증 `[화면별]`

### 7.1 검증 조건
- 급여대장의 **귀속연월을 수정**할 때 (급여대장 수정 팝업 HRIS_PR_03에서 수정)
- 해당 급여대장에 **이미 추가된 임직원** 중 **변경된 귀속연월에 근무하지 않은 임직원**이 존재하는 경우

### 7.2 confirm 노출
- 문구:
  ```
  선택한 귀속연월에 근무하지 않은 근로자가 포함되어 있습니다.
  해당 근로자를 삭제 후 급여대장을 수정하시겠습니까?
  ```
- 문구 하단에 **해당 임직원 리스트** 표시 `[화면별]`
  - 표시 형식: `성명(사번)`
  - 예시: `홍길동(EMP001), 김철수(EMP002)`

### 7.3 확인/취소 동작
- **확인 버튼** 클릭 시:
  - 해당 임직원들의 급여내역 삭제
  - 급여대장 귀속연월 수정 완료
- **취소 버튼** 클릭 시:
  - 귀속연월 수정 취소 (기존 값 유지)
  - 급여내역 변경 없음

## 8. 충돌 지점 정리

| # | 항목 | 공통 정책 | 화면별 정책 | 적용 결과 |
|---|------|-----------|-------------|-----------|
| ① | 확정 상태 수정 팝업 | 수정 팝업: 편집 가능 + 취소/저장 버튼 + confirm | 확정 시: 읽기 전용 + 닫기 버튼만 + confirm 미노출 | **화면별 정책 우선 적용** - 확정 상태에서는 읽기 전용 모드로 표시 |
| ② | 급여항목 관리 팝업 리스트 정렬 | 최근 등록 순 | 등록 오래된 순 | **화면별 정책 우선 적용** - 급여항목 관리에서 등록 오래된 순서로 제공 |

> - 충돌 ①: 공통 정책의 수정 팝업은 항상 편집 가능하지만, 급여확정 상태에서는 읽기 전용 + 닫기 버튼만 제공하여 이탈 방지 confirm도 생략됩니다.
> - 충돌 ②: 공통 정책의 기본 정렬은 "최근 등록 순"이지만, 급여항목 관리 팝업에서는 등록 오래된 순으로 제공합니다.
