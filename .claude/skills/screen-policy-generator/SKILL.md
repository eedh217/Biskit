---

# 목표

- 사용자가 기능 설명을 제공하면,
  1) common-ui-policy를 기본값으로 적용하고
  2) 화면별 요구사항을 병합하고
  3) 충돌 지점을 명시하고
  4) 질문 목록을 분리하고
  5) policy_xxx_merged.md 형태의 완성된 정책서를 생성한다.

- 산출물은 항상 "최종 정책서 형태의 MD 파일"이다.
- 출력은 구현이 아니라 "정책 문서"다.

---

# 입력

사용자는 아래 중 하나 형태로 입력할 수 있다:

1. 기능 설명 자연어
2. 기존 policy_xxx.md 파일
3. 기존 policy_xxx.md + 추가 요구사항
4. 일부 필드 정의만 제공

---

# 기본 동작 규칙

## 1. 공통 정책 자동 적용

- common-ui-policy를 기본값으로 반드시 적용한다.
- 사용자가 별도로 언급하지 않아도 다음은 자동 포함:
  - 리스트 페이지 사이즈 (15/30/50/100, 기본 30)
  - 엑셀 다운로드 3가지 범위 규칙
  - 선택삭제 confirm
  - 추가/수정 팝업 confirm
  - 토스트 1.5초
  - blur/submit validation 규칙
  - 서버 에러 alert 정책

---

## 2. 화면별 정책 병합

- 화면별 요구사항은 공통 정책 위에 덧붙인다.
- 공통 정책과 충돌 시:
  - 화면별 정책 우선
  - 충돌 지점 섹션을 별도로 생성

# 질문 생성 규칙

다음이 없으면 반드시 질문 생성:

- 검색항목 정의 없음
- 삭제 정책 불명확
- 중복 체크 기준 없음
- 필수/선택 구분 없음
- 키값 전달 방식 미정

---

# 출력 규칙

- 항상 완성형 정책서로 출력
- 구현 코드 작성 금지
- UI 디자인 제안 금지
- 테스트케이스 생성 금지
- 문서 구조 깨지지 않도록 유지

# 파일명 규칙

- 기본 파일명:
  policy_[기능명]_merged.md

- 기능명이 명확하지 않으면:
  policy_screen_merged.md
